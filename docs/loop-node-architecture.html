LoopNode 循环节点架构

DAG 引擎中的循环控制方案设计与演示

<\!-- Section 1: 重构前 vs 重构后 -->
📊 重构前 vs 重构后对比
旧方案
静态展开

6 个静态节点硬编码 2 轮循环

supervisor_review_1 → final_revision_1 → supervisor_review_2 → final_revision_2
  • 轮次固定，无法动态调整
  • 字符串匹配脆弱，易出错
  • 无耗尽策略，超限后行为未定义
新方案
LoopNode 循环

1 个 LoopNode 封装循环逻辑

review_loop
review → revision ↺
  • 轮次可配置（MaxRounds）
  • JSON 结构化解析，类型安全
  • 可配置耗尽策略（use_last / error）

<\!-- Section 3: 执行时序 -->
⏱ LoopNode 执行时序（动画演示）
Round 1
构建子 DAG（review → revision）
创建隔离 State（round_1 命名空间）
执行 review 节点 → 获取审核结果
审核不通过 passed: false
执行 revision 节点 → 生成修订稿
回写结果至 SharedState（review_loop.round_1.*）
Round 2
构建子 DAG（review → revision）
创建隔离 State（round_2 命名空间）
执行 review 节点 → 获取审核结果
审核通过 passed: true
退出循环，写入 meta 信息
