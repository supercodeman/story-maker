<\!-- ===== SECTION 11: 降级与容错 ===== -->

11 / 降级与容错

两层 Fallback 机制

所有 LLM 调用均支持自动降级，确保服务可用性

第一层：同 Provider 内模型降级
qwen-plus (默认)
qwen-turbo
qwen-long

通过 Provider.FallbackModels() 获取降级列表

第二层：跨 Provider 降级
Qwen (主)
Zhipu
DeepSeek
Kimi

当同 Provider 所有模型均失败时触发

IsRetryableError 判定

只有可重试错误才触发降级，其他错误直接返回

429 Rate Limit
Quota Exhausted
Insufficient Balance
500/502/503
Timeout
Connection Refused

各服务降级策略

服务主 Provider降级行为
SuggestionServiceQwen同 Provider 模型降级 → 跨 Provider
UserPreferenceServiceQwen同 Provider 模型降级 → 跨 Provider → 规则兜底
Dispatcher (异步任务)配置指定同 Provider → 跨 Provider → 标记 failed
NovelSearchServiceconfig.web_search_model降级到 config.ai_model 离线兜底
